java的Object（equals、hashCode、toString、clone）

Object位于java.lang包，是所有java类的父类，不用明确的写 extends Object。

int hashCode() Returns a hash code value for the object.

protected Object clone() Creates and returns a copy of this object.

本文将会主要阐述equals、hashCode方法以及这两者的关系和toString、clone方法。

equals方法

首先需要明白比较符号 == 的使用：

  1. 比较对象时，判断两对象的引用地址是否相等。
  2. 比较基本数据类型时，比较字面值是否相等。

equals比较的是对象，Object类中的equals方法默认就是只用 == 比较，根据实际情况，只根据引用地址是否相等就来判定两个对象是否相等没有实际意义，我们更新在意的是对象的个个特征（成员变量）是否相等，所以几乎所有的类都要重写equals方法。
java规定重写的equals方法需要实现以下特征：( on non-null object references:)

  1. reflexive(自反性): for any non-null reference value ，x.equals(x)} should return true.
  2. symmetric(对称性): for any non-null reference values x and y, x.equals(y) should return true if and only if y.equals(x)} returns true.
  3. transitive（传递性）: for any non-null reference values x, y, and z, if x.equals(y) returns true and y.equals(z) returns d true, then x.equals(z) should return true.
  4. consistent(一致性): for any non-null reference values x and y, multiple invocations o x.equals(y) consistently return true or consistently return false, provided no information used in equals comparisons on the objects is modified.
  5. For any non-null reference value x, x.equals(null) should return false.

上面的规则设计避免了开发者是调用x.equals(y)还是y.equals(x)的问题。
但是针对于第二条也就是自反性，在使用时要考虑一个问题：两个对象不是一个类时，可能出错。例如manager extends employee，e.equals(m)没有错，因为m属于employee，e有的成员变量m都有，而m有着自身特有的变量，所以在调用m.equals(e)时就会返回false。这就使manager类受到了限制，而这种比较在实际中也是有很实际的意义。所以我们仅在比较之前进行instanceOf比较并不能解决所有的问题。而使用getClass可能会与实际意义不符（例如两个子类之间的成员变量并没什么特殊，可能只是操作不同，这俩子类的对象实际是可以进行equals方法的，也符合自反性，但是getClass就会直接判断这俩对象不等）。 所以针对instanceOf与getClass可以根据具体的情况使用：

  • 如果两个子类中至少有一个有着自己的特殊成员变量，那么为了实现自反性，就必须要使用getClass比较，比如employee与manager。
  • 如果两个子类都没自己再定义自己的成员变量，那么就可以使用instanceOf进行检测。

以下是 java核心技术卷一第十版给出的一个编写完美的equals方法的建议 ：

  1. 显示命名参数otherObject，稍后需要将它转换成另一个叫做other的变量
  2. 检测this与otherObject是否引用同一个对象：
    if (this == otherObject) return if (this == otherObject) return true;
    这条语句只是一个优化，实际上，这是一种经常采用的形式。因为计算这个等式要比一个一个地比较类中的域所付出的代价小得多。
  3. 检测 otherObject 是否为null，如果为null，返回false。这项检测是很有必要的。
    if (otherObject == null) return if (otherObject == null) return false;
  4. 比较this与otherObject是否属于同一个类。如果equls的语义在每个子类中有所改变，就使用getClass检测：
    if (getClass() != otherObject.getClass()) return if (getClass() != otherObject.getClass()) return false;
    如果所有的子类都拥有统一的语义，就是用instanceof检测：
    if (!(otherObject instanceof ClassName)) return if (!(otherObject instanceof ClassName)) return false;
  5. 将otherObject转换为响应类的类型变量：
    ClassName other = (ClassName) otherObject
  6. 现在开始对所有需要比较的域进行比较。使用 == 比较基本类型域，使用equals比较对象域。如果所有的域都匹配，就返回true；否则返回false。
    return filed1 == other.filed1 && Object.equals(filed2, other.filed2) && ,..;
    如果在子类中重新定义equals，就要在其中包含调用 super.equals(other)。
